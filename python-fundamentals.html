Introduction to Python

1. Introduction to Python

Think of a programming language as a way to give instructions to a computer. Some languages are complex and strict, like writing a formal legal document. Python, on the other hand, is like writing a simple, clear recipe. It's designed to be easy to read, write, and understand, which is why it's a perfect first language.

Python is a high-level, interpreted, and general-purpose programming language. Let's break that down:

  • High-level: This means the code you write is closer to human language and less like the complex 1s and 0s the computer actually uses.
  • Interpreted: Python runs your code line by line, much like a translator interpreting a speech as it happens. This makes it easy to test small pieces of code quickly.
  • General-purpose: Python is like a Swiss Army knife. It can be used for almost anything: building websites, analyzing data, automating tasks, and even artificial intelligence.

Why Should You Learn Python?

  • ✅ Simple and Readable: Its clean syntax helps you focus on solving problems, not on memorizing complex rules.
  • ✅ Extremely Versatile: The skills you learn can be applied to many different jobs and industries.
  • ✅ Huge Community: If you ever get stuck, there's a massive global community and countless resources (called libraries) to help you out.

2. Installing and Running Python

To start writing Python, you first need to install it on your computer.

Installation Steps:

  1. Go to the official Python website: python.org.
  2. Navigate to the "Downloads" section. The website will automatically suggest the best version for your operating system (Windows, Mac, etc.).
  3. Run the installer. Important for Windows users: On the first screen of the installer, make sure to check the box that says "Add Python to PATH". This will make it much easier to run Python from anywhere on your command line.

How to Run Python Code

You have two main ways to run Python:

1. Interactive Mode (The Python Shell)

This is like a calculator for Python. It's great for testing single lines of code.

  1. Open your computer's command line (Command Prompt for Windows, Terminal for Mac/Linux).
  2. Type python (or python3 on some systems) and press Enter.
  3. You'll see a >>> prompt. You can now type Python commands directly.
>>> print("Welcome to Revature!")
Welcome to Revature!
>>> 4 + 5
9

2. Script Mode (Running .py Files)

This is how you'll write most of your programs.

  1. Open a simple text editor (like Notepad, VS Code, etc.).
  2. Save the file with a .py extension (e.g., my_first_program.py).
  3. Navigate to that file's directory in your command line.
  4. Run the file by typing python my_first_program.py.

Example my_first_program.py file:

# This is a program to greet a new student.
student_name = "Alex"
print("Hello, " + student_name + "! Get ready to learn.")

3. Python Syntax vs. Java Syntax

If you have seen other languages like Java, you'll notice Python is much less "wordy." It prioritizes simplicity. The main goal is readability.

Here's a quick comparison:

Task Java (More Verbose) Python (Simple & Clean)
Printing "Hello" System.out.println("Hello"); print("Hello")
Declaring a Variable String name = "Maria"; name = "Maria"
An if statement if (x > 10) { ... } if x > 10: ...

Key Differences to Remember:

  • No Semicolons: Python does not require a semicolon (;) at the end of each line.
  • Dynamic Typing: You don't have to declare the type of a variable (like String or int). Python figures it out automatically.
  • Indentation is Crucial: This is the most important rule! Python uses whitespace (spaces or tabs) to define code blocks (like loops or if statements) instead of curly braces {}. Indent your code consistently, or it will not run.

4. Data Types and Variables

What is a Variable? A variable is like a labelled box where you can give the box a name, and you can change what's inside it later.

# Here, 'batch_name' is the variable (the box label)
# and "PEP-October-2025" is the value (the info inside the box).
batch_name = "PEP-October-2025"

What are Data Types? A data type is the category of information you are storing. Python has several built-in types. Here are the most common ones you'll use:

  • String (str): Used for text. You must enclose it in single quotes (') or double quotes (").
    • student_name = "John Doe"
    • location = "Mumbai"
  • Integer (int): Used for whole numbers (no decimals).
    • student_count = 30
    • score = 95
  • Float (float): Used for numbers that have a decimal point.
    • average_grade = 88.5
    • pi = 3.14
  • Boolean (bool): Used to represent one of two values: True or False. It's essential for logic and making decisions in your code. (Note the capital T and F).
    • is_enrolled = True
    • has_passed_assessment = False

You can see these in action all at once:

# A program to store info about a Revature student
student_id = 1024       # Integer
student_name = "Sarah"  # String
gpa = 3.8               # Float
is_active_student = True # Boolean

# You can print them out to see their values
print(student_name)
print(gpa)

Core Programming Concepts

1. Operators and Expressions

Think of operators as the action words or symbols in your code. They perform operations on your variables and values. An expression is a combination of values, variables, and operators that Python evaluates to produce a result.

For example, total_score = 90 + 5 is an expression. The + is an operator.

Key Types of Operators:

  • Arithmetic Operators: These are for performing mathematical calculations.
    • + (Addition)
    • - (Subtraction)
    • * (Multiplication)
    • / (Division)
    • % (Modulus - gives the remainder of a division)
    • ** (Exponent - raises to the power of)
num1 = 10
num2 = 3
print(num1 % num2) # Output: 1 (because 10 divided by 3 has a remainder of 1)
print(num2 ** 4) # Output: 81 (because 3 * 3 * 3 * 3 = 81)
  • Comparison Operators: These are used to compare two values. The result is always a Boolean (True or False).
    • == (Equal to)
    • != (Not equal to)
    • > (Greater than)
    • < (Less than)
    • >= (Greater than or equal to)
    • <= (Less than or equal to)
passing_score = 75
student_score = 82
print(student_score > passing_score) # Output: True
  • Logical Operators: These are used to combine conditional statements.
    • and: Returns True only if both statements are true.
    • or: Returns True if at least one of the statements is true.
    • not: Reverses the result, returning False if the result is true, and vice-versa.
attendance = 90
score = 85
# The student passes if attendance is over 80 AND score is over 75
print(attendance > 80 and score > 75) # Output: True

2. Control Flow Statements (Making Decisions)

Control flow statements are like a GPS for your program. They direct the flow of execution based on certain conditions, allowing your code to make decisions and repeat actions.

if, elif, else Statements:

This structure lets your program choose a path. "If a condition is true, do this. Else if another condition is true, do something else. Otherwise, do a default action."

  • if: The block of code that runs if the condition is True.
  • elif: (short for "else if") An optional block to check another condition if the first if was False.
  • else: An optional block that runs if all preceding if and elif conditions were False.
# A program to assign a grade based on a score
score = 88

if score >= 90:
    grade = "A"
elif score >= 80: # This runs because score is not >= 90, but is >= 80
    grade = "B"
elif score >= 70:
    grade = "C"
else:
    grade = "Needs Improvement"
    
print(f"The student's grade is: {grade}") # Output: The student's grade is: B

Loops (Repeating Actions):

Loops are used to execute a block of code multiple times.

  • for loop: Executes code for each item in a sequence (like a list). It's best used when you know how many times you want to loop.
  • while loop: Repeats a block of code as long as a condition is True. It's best used when you don't know how many times you need to loop. Warning: Be careful! If the condition never becomes False, you will create an infinite loop.
# Print a welcome message for each student in the batch
students = ["Priya", "Rohan", "Sana"]

for student in students:
    print(f"Welcome to the Revature PEP program, {student}!")
# A simple countdown
count = 5

while count > 0:
    print(count)
    count = count - 1 # This line is crucial to prevent an infinite loop!
    
print("Liftoff!")

3. Functions in Python

A function is a reusable block of code that performs a specific task. Think of it like a recipe. You define the recipe once, and then you can "call" it anytime you want to use it, without having to rewrite all the steps. This makes your code organized, efficient, and easy to read. This follows the DRY (Don't Repeat Yourself) principle.

Defining and Calling a Function:

  • You define a function using the def keyword.
  • Parameters are the variables listed inside the parentheses in the function definition (the "ingredients" for your recipe).
  • Arguments are the actual values you send to the function when you call it.
# DEFINE the function (the recipe)
def greet_student(name, batch):
    """This function prints a personalized greeting."""
    print(f"Hello, {name}! Welcome to the {batch} batch.")
# CALL the function (using the recipe)
greet_student("Aarav", "Python PEP") # "Aarav" and "Python PEP" are arguments
greet_student("Meera", "Java PEP")

Returning a Value:

Some functions just perform an action (like print), while others calculate something and give you back a result. The return keyword is used to send a value back from the function.

def calculate_area(length, width):
    """This function calculates the area of a rectangle and returns the result."""
    area = length * width
    return area

# Call the function and store the returned value in a variable
room_area = calculate_area(10, 5)

print(f"The area is: {room_area}") # Output: The area is: 50

4. Data Structures: Lists, Tuples, Sets, & Dictionaries

Data structures are special containers for organizing and storing groups of related data.

  • List []:
    • What it is: An ordered and changeable collection of items.
    • Analogy: A grocery list. You can add items, remove them, and the order matters.
    • Use it when: You need a collection of items that might change.
technologies = ["Python", "Java", "SQL"]
technologies.append("AWS") # Add an item to the end
print(technologies[0])     # Access the first item (indexing starts at 0)
# Output: Python
print(technologies)
# Output: ['Python', 'Java', 'SQL', 'AWS']
  • Tuple ():
    • What it is: An ordered and unchangeable collection of items.
    • Analogy: The days of the week. They have a fixed order and never change.
    • Use it when: You have data that should not be modified after it's created.
primary_colors = ("Red", "Green", "Blue")
print(primary_colors[1]) # Accessing works just like lists
# Output: Green
# primary_colors.append("Yellow") # This would cause an error!
  • Set {}:
    • What it is: An unordered collection of unique items. Duplicates are automatically removed.
    • Analogy: The unique ingredients required for a recipe. Order doesn't matter, and you only list each ingredient once.
    • Use it when: You need to store unique items and don't care about the order.
student_skills = {"Python", "SQL", "Git", "Python"}
print(student_skills) # Notice "Python" only appears once
# Output: {'SQL', 'Git', 'Python'} (order may vary)
  • Dictionary {}:
    • What it is: A collection of key-value pairs. Each value is accessed by its unique key.
    • Analogy: A real dictionary. You look up a key (the word) to find its value (the definition).
    • Use it when: You need to store associated data, like user profiles or configurations.
revature_student = {
    "name": "Anjali",
    "id": 4096,
    "is_active": True
}
# Access a value by its key
print(revature_student["name"])
# Output: Anjali

5. Classes and Objects

Object-Oriented Programming (OOP) is a way of structuring your code to be more organized, reusable, and easier to manage, especially for large projects.

The main idea is to bundle related data (properties) and behaviors (methods) into individual "objects."

    Analogy: Think of building with LEGOs. Instead of having one giant, messy pile of bricks, you first design different types of LEGO pieces (Classes). Then, you use those designs to create many individual pieces (Objects) that you can connect to build something amazing.
  • Class: The blueprint. A class is a blueprint for creating objects. It defines the properties and behaviors that all objects of that type will have.
    • class Student: - This is the blueprint for a "Student".
  • Properties (Attributes): What a student has (e.g., name, batch).
  • Behaviors (Methods): What a student can do (e.g., introduce_self()).
  • Object: The actual instance. An object is a specific instance created from a class. You can create many unique objects from a single class blueprint.
    • student1 = Student("Ravi", "Java") - This creates an actual student object named Ravi.
    • student2 = Student("Priya", "Python") - This creates another, separate student object named Priya.

Creating and Using a Class

Let's see this in code.

# This is the blueprint for a Revature student
class RevatureStudent:
    
    # The __init__ method is the "constructor". It runs automatically
    # when you create a new object from this class.
    # "self" refers to the specific object being created.
    def __init__(self, student_name, student_batch):
        # These are the properties (attributes)
        self.name = student_name
        self.batch = student_batch
        
        print(f"A new student named {self.name} has been created!")

    # This is a behavior (method)
    def introduce_self(self):
        print(f"Hello, my name is {self.name} and I am in the {self.batch} batch.")
        
    # --- Using the blueprint to create actual objects ---

    # Create the first student object (an instance of the class)
    student1 = RevatureStudent("Ravi Kumar", "Java PEP")
    
    # Create a second student object
    student2 = RevatureStudent("Priya Singh", "Python PEP")
    
    # Now, let's call their methods. Each object has its own data.
    student1.introduce_self() 
    # Output: Hi, my name is Ravi Kumar and I am in the Java PEP batch.
    
    student2.introduce_self()
    # Output: Hi, my name is Priya Singh and I am in the Python PEP batch.

2. Inheritance

Inheritance allows a new class (the "child" class) to receive—or inherit—all the properties and methods from an existing class (the "parent" class). This is a powerful way to reuse code and establish an "is-a" relationship (e.g., a Developer is an Employee).

  • Analogy: A child inherits traits from a parent. The child has all the parent's general characteristics (like having a name) but can also have their own unique ones (like a favorite programming language).
# PARENT class: Defines general properties and behaviors
class Employee:
    
    def __init__(self, name, employee_id):
        self.name = name
        self.employee_id = employee_id
        
    def show_details(self):
        print(f"ID: {self.employee_id}, Name: {self.name}")

# CHILD class: Inherits from Employee
class Developer(Employee):
    
    def __init__(self, name, employee_id, language):
        # 'super()' calls the parent's constructor to set up the inherited parts
        super().__init__(name, employee_id)
        
        # Add a new property specific to the Developer
        self.language = language

# Create a Developer object
dev1 = Developer("Sanjay", "REV123", "Python")

# The 'dev1' object can use the 'show_details' method from the Employee class
# because it inherited it!
dev1.show_details() # Output: ID: REV123, Name: Sanjay

3. Polymorphism

Polymorphism (from Greek, meaning "many forms") is the principle that allows different objects to respond to the same method call in their own, unique ways. This is typically achieved through method overriding, where a child class provides a specific implementation for a method that it inherited from its parent class.

  • Analogy: Think of a "start" button. On a car, it starts the engine. On a microwave, it starts cooking. The interface (the button) is the same, but the action it performs is specific to the object.
# Using the same parent Employee class from before...

# --- Child classes OVERRIDING the parent's method ---

class Developer(Employee):
    def __init__(self, name, employee_id, language):
        super().__init__(name, employee_id)
        self.language = language
        
    # Override the show_details method for a more specific output
    def show_details(self):
        print(f"ID: {self.employee_id}, Name: {self.name}, Role: Developer, Language: {self.language}")

class Manager(Employee):
    def __init__(self, name, employee_id, team_size):
        super().__init__(name, employee_id)
        self.team_size = team_size

    # Override the show_details method here as well
    def show_details(self):
        print(f"ID: {self.employee_id}, Name: {self.name}, Role: Manager, Team Size: {self.team_size}")
# --- Polymorphism in Action ---
dev1 = Developer("Sanjay", "REV123", "Python")
Mgr1 = Manager("Anjali", "REV007", 12)

# Calling the SAME method name on different objects produces different results
dev1.show_details()
Mgr1.show_details()

3. Encapsulation

Encapsulation is the practice of bundling data (attributes) and the methods that operate on that data into a single unit, a class. A key part of this is data hiding: protecting an object's internal data from being accessed or modified directly from outside. Instead, access is controlled through public methods.

  • Analogy: Think of a medicinal capsule. The capsule (the class) holds all the necessary ingredients (the data) together. You don't interact with the powders inside directly; you use the simple interface of swallowing the capsule (calling a method) to get its effect. This prevents accidental misuse of the ingredients.

In Python, we use a naming convention to indicate that an attribute should be protected. A double underscore __ prefix makes an attribute "private," making it difficult to access from outside the class.

class BankAccount:
    
    def __init__(self, initial_amount):
        # By using '__balance', we make this attribute private.
        self.__balance = initial_amount
        
    # Public method to deposit money
    def deposit(self, amount):
        if amount > 0:
            self.__balance += amount
            print(f"Deposited ${amount}. New balance is ${self.__balance}.")
        else:
            print("Deposit amount must be positive.")
            
    # Public method to get the balance (controlled access)
    def get_balance(self):
        return self.__balance

# Create an account
my_account = BankAccount(1000)

# You CAN'T access the private attribute directly
# print(my_account.__balance) # This will cause an AttributeError

# You MUST use the public methods to interact with the data
my_account.deposit(500) # Output: Deposited $500. New balance is $1500.
print(f"Current balance: ${my_account.get_balance()}") # Output: Current balance: $1500

4. Abstraction

Abstraction means hiding complex implementation details and only showing the essential features to the user. It simplifies complex systems by providing a clear and simple interface to work with.

  • Analogy: When you drive a car, you use a simple interface—the steering wheel, pedals, and gear stick. You don't need to know the complex mechanics of the engine or the transmission system to operate the car. The complexity is abstracted away.

In Python, this is often implemented using Abstract Base Classes (ABCs). An abstract class defines a common interface that its child classes are required to implement. You cannot create an object from an abstract class itself.

from abc import ABC, abstractmethod

# Create an Abstract Base Class for any kind of 'File'
class File(ABC):
    
    def __init__(self, name):
        self.name = name
    
    # This is an abstract method. Any child class MUST implement it.
    @abstractmethod
    def open(self):
        pass

# Create concrete child classes that inherit from File
class PdfFile(File):
    
    # Provide a specific implementation for the 'open' method
    def open(self):
        print(f"Opening PDF file: {self.name}.pdf")

class TxtFile(File):
    
    # Provide another specific implementation for the 'open' method
    def open(self):
        print(f"Opening text file: {self.name}.txt")
        document1 = PdfFile("Revature-Handbook")
document2 = TxtFile("Meeting-Notes")

# We can use the same simple 'open()' interface for different file types
document1.open() # Output: Opening PDF file: Revature-Handbook.pdf
document2.open() # Output: Opening text file: Meeting-Notes.txt

3. Exception Handling

An exception is an error that occurs during the execution of a program. If not handled, it will cause the program to crash. Exception handling is a way to gracefully manage these errors so your program can continue running or terminate smoothly.

  • Analogy: Exception handling is a safety net. If your program's "acrobat" (the code) slips and falls (an error occurs), the safety net catches it and prevents a disaster (a crash).

We use the try...except block to handle exceptions.

  • try: You place the code that might cause an error inside the try block.
  • except: If an error does occur in the try block, the code inside the except block is executed. The program does not crash.
  • finally: (Optional) This block of code will run no matter what, whether an exception occurred or not. It's often used for cleanup tasks.
# A program to calculate a student's score percentage
try:
    total_marks = float(input("Enter total marks: "))
    marks_obtained = float(input("Enter marks obtained: "))
    
    # This line could cause a ZeroDivisionError
    percentage = (marks_obtained / total_marks) * 100
    print(f"Percentage: {percentage:.2f}%")

# This block runs ONLY if the user enters 0 for total_marks
except ZeroDivisionError:
    print("Error: Total marks cannot be zero. Please try again.")

# This block runs ONLY if the user enters text instead of a number
except ValueError:
    print("Error: Please enter valid numbers, not text.")
    
# This block runs no matter what
finally:
    print("--- End of calculation ---")
