Java Practice Lab:

Mentor's Note:

  • Don't rush. Type the code yourself — don't copy paste.
  • Every error you see teaches you something:
  • a missing bracket, a misspelled variable, a type mismatch — all are clues, not failures.

Step 0: Make sure Java is ready

  • JDK installed?
  • Can you run java -version and javac -version in your terminal?
  • Great! You're ready to start coding.

Step 1: Create your first class

Think of a class as a container for your program.

class HelloWorld { // we will put code here next }

Step 2: Add main() method

This is where Java starts running your program.

public static void main(String[] args) { System.out.println("Hello, World!"); }

Try this:

  • Add the main() method inside your class.
  • Run it. You should see: Output: "Hello world!" — in the Console

Step 3: Play with print statements

  • Try printing your name, age, and favorite hobby.
  • Use both System.out.println() and System.out.print().
System.out.println("Name: Sunny"); System.out.println("Age: 21"); System.out.print("Hobby: Coding"); System.out.print(" & Reading");

Mini Challenge:

  • Create a class: PersonalInfo.
  • In main(), declare variables: name (String), age (int), height (double).
  • Print a message like:

Target Output: Hi, I am Sunny, I am 21 years old and 5.8 ft tall.

Step 5: Explore & Experiment

  • What happens if you forget public static void? Try it!
  • What if you remove a semicolon? What does the error message say?

Goal: Learn by trying, breaking, and fixing your code.

Java Practice Lab: Variables & Data Types

You've already learned how to create a class, write a main() method, and print text to the console. Now let's make your code a little smarter by using variables — small storage boxes for your program's data.

Challenge 1: Let's Store Something!

Create a class called — MyProfile

Inside it, store the following details as variables:

  • Your name (String)
  • Your age (int)
  • Your average score (double)
  • Whether you are a student or not (boolean)

Then print them all in one meaningful sentence.

Hint:
Use + to join your text with variable values.

Think about this:
If you wrap numbers in quotes, they become text. What's the difference when printing "21" vs 21?

Expected Output Example:
Hi, my name is Arjun, I am 21 years old, my average score is 89.5, and it's true I am a student!

Pitfalls to watch for:

  • Missing semicolon — Java won't forgive you!
  • Wrong data type — like writing int name = "Sunny"; X
  • Boolean prints as true or false — no need for quotes.

Done? Great!
You just used data types and string concatenation to describe yourself.

Challenge 2: Quick Math

Create a new class called — SimpleMath.
This time, declare two integer variables — maybe marks of two subjects or prices of two items.
Find and print:

  • Sum
  • Difference
  • Product
  • Division

Hint:
Use +, -, *, /.
Print each result clearly — like "Sum: " before the value.

Expected Output Example:
Sum: 30
Difference: 10
Product: 200
Division: 2

Watch out for:

  • Division between integers gives whole numbers only!
  • If you want decimal output, use double.

Challenge 3: Access in Action

You've been writing everything inside main().
Now try something slightly more organized.
Create a class StudentDetails and add:

  • A variable name with private access.
  • A method displayDetails() that prints the name.

In your main() method, create an object of the class and call that method.

Hint:
This is your first taste of OOP (Object-Oriented Programming) — methods belong to classes, and you call them through objects.

Expected Output Example:
Student name: Sunny

Pitfalls:

  • Forgetting public in the class or method may stop access.
  • Don't write code outside of methods — Java doesn't allow loose statements.

Nice work!
You just practiced access modifiers, methods, and variables — that's object-oriented programming in action.

Conditionals (if / else)

So far, your programs just printed fixed results.
Now it's time to make them react — to make choices.

If something is true, do this.
Otherwise, do that.

Challenge 1: Simple Decision

Create a class called AgeChecker.

Ask Java to print whether a person is an adult or not.

int age = 17;

If age is 18 or above, print "You are an adult."
Otherwise, print "You are not an adult yet."

Expected Output Example:
You are not an adult yet.

Hint:
Use if (condition) { ... } else { ... }

Pitfalls:

  • Forgetting parentheses around the condition.
  • Missing { } — especially if you add multiple lines inside.

Challenge 2: Add More Conditions

Make a class GradeSystem.

Declare:

int marks = 82;

Print:

  • "Excellent" if marks ≥ 90
  • "Good" if marks ≥ 75
  • "Average" if marks ≥ 50
  • "Fail" otherwise

Expected Output Example:
Good

Hint:
Use if... else if... else
Try changing marks to test different outcomes.

Challenge 3: Nested Decisions

Let's combine what you know!
Create a class called EligibilityCheck.

Declare:

int age = 20; boolean hasLicense = true;

If age is above 18, check inside whether they have a license.
If yes → print "You can drive!"
If no → print "You're old enough, but you need a license!"
Otherwise → print "You're too young to drive!"

Expected Output Example:
You can drive!

Pitfalls:

  • Forgetting to close nested braces { }
  • Misplacing else — it always pairs with the closest if.

Inheritance & Polymorphism

You already know how to create classes, objects, fields, methods, and constructors.
Now let's explore how one class can reuse another class's code, and how methods can behave differently.

Challenge 1: Basic Inheritance

Goal: Create a Person class with name and a method displayInfo().

  • Create a Student class that extends Person.
  • Add rollNumber to Student.
  • Print both name and roll number from a Student object.

Hints:

  • Use extends keyword to inherit.
  • The child class automatically gets the parent's fields and methods.
  • Create an object of the child class to test.

Expected Output Example:
Name: Sunny
Roll Number: 101

Pitfalls:

  • Forgetting extends → child class won't inherit.
  • Parent method still accessible, no object needed for parent class.

Challenge 2: Compile-time Polymorphism (Method Overloading)

Goal: In Student class, create multiple display() methods:

  • display() → no parameters
  • display(String subject) → prints subject info

Hints:

  • Same method name, different parameter types or count.
  • This is called compile-time polymorphism.

Expected Output Example:
Displaying student info
Displaying student info for Math

Pitfalls:

  • Changing only return type does not count as overloading.
  • Parameters must differ in number or type.

Challenge 3: Runtime Polymorphism (Method Overriding)

Goal:

  • In Person class, have displayInfo() print name.
  • In Student class, override displayInfo() to print name and roll number.

Hints:

  • Same method name, same parameters, in child class.
  • Use a Person reference pointing to a Student object.

Expected Output Example:
Name: Sunny
Roll Number: 101

Pitfalls:

  • Forget @Override annotation → still works but annotation helps detect mistakes.
  • If method is private/final/static, it cannot be overridden.

Challenge 4: Multiple Inheritance Note

Concept:

  • Java does not allow multiple inheritance with classes.
  • Example: class Student extends Person, Teacher X → compilation error.

Hints:

  • Why? Diamond problem (ambiguity in inherited methods).
  • Solution: Use interfaces if you need multiple "parents".

Expected Output Example:

  • Try creating a class with two parent classes → compiler will show error.
  • Instead, implement multiple interfaces.

    • Mini Challenge: Combine Inheritance & Polymorphism

    Goal:

    • Create a Teacher class extending Person.
    • Both Student and Teacher override displayInfo().
    • Create a Person array containing Student and Teacher objects.
    • Loop through array and call displayInfo().

    Expected Output Example:
    Student: Sunny, Roll Number: 101
    Teacher: Mr. Raj, Subject: Math

    Hints:

    • Use polymorphic reference: Person p = new Student();
    • Loops + polymorphism = powerful combo.

    Encapsulation & Abstraction

    You've already learned classes, objects, inheritance, and polymorphism.
    Now let's make your classes secure, organized, and easy to use.

    Challenge 1: Read-Only Field

    Goal: Create a class BankAccount.

    • Field: accountNumber → private
    • Provide only a getter (no setter).
    • Print the account number in main().

    Hints:

    • Getter method = public int getAccountNumber()
    • Direct access like accountNumber outside class → X

    Expected Output Example:
    Account Number: 12345

    Pitfalls:

    • Forgetting private → defeats encapsulation.
    • Writing a setter here will make it read-write, not read-only.

    Challenge 2: Write-Only Field

    Goal:

    • Field: password → private
    • Provide only a setter.
    • Try setting the password and see that it cannot be read.

    Hints:

    • Setter method = public void setPassword(String pwd)
    • Attempting System.out.println(password) outside class → X

    Expected Output Example:
    Password set successfully!

    Challenge 3: Read/Write Field

    Goal:

    • Field: balance → private
    • Provide both getter and setter.
    • Update balance and print the new balance.

    Expected Output Example:
    Initial Balance: 1000
    Updated Balance: 1500

    Hints:

    • Use getter to read.
    • Use setter to write.
    • Keep logic safe, e.g., prevent negative balance.

    Challenge 4: Abstract Class

    Goal:

    • Create abstract class Vehicle.
    • Abstract method: start().
    • Create concrete class Car extending Vehicle and implement start().
    • Call start() from a Car object.

    Hints:

    • Abstract class cannot be instantiated.
    • Must implement all abstract methods in child class.

    Expected Output Example:
    Car is starting

    Challenge 5: Interface Implementation

    Goal:

    • Create interface Playable with method play().
    • Create class Guitar implementing Playable.
    • Implement play() to print a message.
    • Call play() from Guitar object.

    Expected Output Example:
    Playing the guitar

    Hints:

    • Interfaces are fully abstract (no method body in interface).
    • Class can implement multiple interfaces → solves multiple inheritance problem.

    Mini Exploration Challenge: Combine Encapsulation & Abstraction

    Goal:

    • Create abstract class Employee with abstract method calculateSalary().
    • Field: employeeName (private, read-only).
    • Concrete class Manager extends Employee.
    • Implement calculateSalary() and print employee name + salary.

    Expected Output Example:
    Employee: Rajesh
    Salary: 50000

    Exception Handling

    Errors happen. Even in real-world programs, something unexpected can occur — like dividing by zero or opening a missing file.
    Java lets you handle errors gracefully using exceptions.

    Challenge 1: Basic try-catch

    Goal: Handle division by zero.

    • Divide an integer by 0 inside a try block.
    • Catch the exception and print a friendly message.

    Hints:

    • Use try { ... } catch (ArithmeticException e) { ... }
    • Print e.getMessage() to see the error.

    Expected Output Example:
    Cannot divide by zero!

    Pitfalls:

    • Forgetting the catch block → program will crash.
    • Catching the wrong exception type → won't handle the error.

    Challenge 2: Multiple Catch Blocks

    Goal: Handle multiple possible errors.

    • Try dividing two numbers (ArithmeticException).
    • Try accessing an array element out of bounds (ArrayIndexOutOfBoundsException).

    Hints:

    • Use multiple catch blocks in order.
    • Each block handles a specific exception type.

    Expected Output Example:
    Cannot divide by zero!
    Array index is out of range!

    Challenge 3: Finally Block

    Goal: Always execute a block whether an exception occurs or not.

    • Use try-catch-finally.
    • In finally, print "Execution finished!"

    Expected Output Example:
    Cannot divide by zero!
    Execution finished!

    Goal: Handle exceptions by throwing them.

    • Create a method checkAge(int age).
    • Throw an IllegalArgumentException if age < 18.
    • Call it from main() and catch it.

    Hints:

    • throw new IllegalArgumentException("Age must be 18 or above")
    • Method signature can have throws IllegalArgumentException

    Goal: See the difference.

    • Unchecked: ArithmeticException, NullPointerException → no compile-time requirement to handle.

    Hints:

    • Try opening a file that doesn't exist → compiler will force you to handle.
    • Try dividing by zero → compiler won't complain, but runtime exception occurs.

    Expected Output Example:
    File not found! Please check the path.
    Cannot divide by zero!
